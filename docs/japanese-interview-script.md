## 30 秒项目介绍（日文）

**日文：**

個人で「留学・海外生活 情報整理プラットフォリオ」を開発しました。Next.js と TypeScript を使い、記事整理、`/plan` のルールベース進路提案、mock の `/ai` 問答 MVP、多言語の Case Study ページを実装しています。商用サービスではなく、フロントエンド実装とプロダクト設計力を示すための作品です。

**中文大意：** 个人开发了留学信息整理作品集，用 Next.js 和 TypeScript 做了文章、规则版 /plan、mock /ai 和多语言 Case Study，不是商业产品，用来展示前端和产品能力。

## 1 分钟项目介绍（日文）

### 1. 项目背景

**日文：**

留学を考える人は、国選び、予算、語学、ビザなど情報が分散しやすいです。それを一つのポートフォリオとして、ページとフローに整理しました。

**中文大意：** 留学信息很分散，我把它们整理成页面和流程放进作品集。

### 2. 做了什么

**日文：**

記事システム、`/plan` の進路プランニング、`/ai` のローカル問答 MVP、`/case-study` のプロジェクト説明ページを作りました。Case Study から全体像を説明できるようにしています。

**中文大意：** 做了文章、/plan、/ai、/case-study，可以从 Case Study 讲清楚项目。

### 3. 技术栈

**日文：**

Next.js、React、TypeScript、Tailwind CSS を使い、GitHub と Vercel で公開しています。ロジックは `lib/plan` や `lib/ai` に分離しています。

**中文大意：** Next.js 全家桶 + GitHub/Vercel，逻辑放在 lib 里。

### 4. 学到了什么

**日文：**

限られたスコープでプロダクトを分解し、説明可能なルール設計と AI MVP の境界設計を意識して開発しました。

**中文大意：** 在有限范围内做产品拆解，重视可解释规则和 AI 边界设计。

## 3 分钟项目介绍（日文）

### 1. 为什么做这个项目

**日文：**

日本で IT 就職を目指す中で、単なるチュートリアルではなく、自分で要件から設計したポートフォリオが必要だと考えました。留学・海外生活は情報量が多く、プロダクト分解の題材として適していると判断しました。

**中文大意：** 为了日本 IT 求职，需要自己做需求设计的作品集，留学信息量大，适合练产品拆解。

### 2. 主要功能

**日文：**

四つの柱があります。記事で情報入口を作り、`/plan` で予算・語学・目標・リスクからルート提案を出し、`/ai` で mock による構造化問答を示し、`/case-study` で背景・技術判断・担当範囲を説明します。中文 / 日本語 / English に対応しています。

**中文大意：** 四块：文章入口、/plan 规则推荐、/ai mock 问答、/case-study 面试讲解，三语言。

### 3. 技术实现

**日文：**

Next.js App Router でページを分け、UI は components、ビジネスロジックは lib に置いています。`/plan` は route-engine、scoring-engine などに分割し、面接で入力から出力まで説明できます。`/ai` は本番 API を使わず、回答構造と UX を先に固定しました。

**中文大意：** App Router + 组件/lib 分离，/plan 引擎可讲解，/ai 先固定结构不接 API。

### 4. 边界设计

**日文：**

DB、ログイン、課金はありません。本番 AI も未接続です。ビザや就業の保証もしません。情報整理とポートフォリオ展示に限定し、誤解を避けるよう Case Study と README で明示しています。

**中文大意：** 无 DB/登录/付费/真实 AI/签证就业承诺，边界在 Case Study 和 README 写清楚。

### 5. 后续计划

**日文：**

優先順位は、テスト追加、コンテンツ拡充、英日 README の整備、`/plan` と `/ai` の連携強化です。本番 AI を接続する場合は、免責事項と安全設計を先に行います。

**中文大意：** 先测试、内容、README、联动；接真实 AI 前先做安全设计。
